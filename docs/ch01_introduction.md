# Глава 1 — Введение

## Что такое Orbitron?

**Orbitron** — это компилируемый, статически типизированный язык программирования,
созданный с одной целью: объединить лучшие идеи самых популярных и выразительных
языков мира в единый, цельный, практичный язык.

Orbitron компилирует исходные файлы `.ot` в один из двух форматов:

- **Нативные бинарные файлы** через LLVM IR (та же инфраструктура, что использует Clang, Rust и Swift)
- **Байт-код JVM** (файлы `.jar`), которые работают везде, где есть Java

Вы получаете производительность компилируемого кода, выразительность современных
скриптовых языков и плавную кривую обучения для разработчиков с любым бэкграундом.

## Философия проектирования

Orbitron следует нескольким ключевым принципам:

### 1. Знакомость важнее новизны

Orbitron заимствует синтаксис, который программисты уже знают и любят:

| Возможность | Источник вдохновения |
|-------------|---------------------|
| Переменные, `func`, типы | Go, Rust |
| `struct + impl` | Go, Rust |
| `class + init` | Java, Kotlin, C# |
| `const` | Rust, C++ |
| Оператор `**` | Python |
| Оператор `\|>` | Elixir, F# |
| `unless` | Ruby |
| `$"..."` | C#, Kotlin |
| `defer` | Go |
| `repeat N` | Lua, Pascal |
| `? :` тернарный | C, Java |

Вместо того чтобы придумывать новый синтаксис для каждой возможности,
Orbitron использует то, что программисты уже умеют. С первой же программы
вы будете чувствовать себя как дома.

### 2. Два стиля ООП

Orbitron предлагает два стиля объектно-ориентированного программирования:

```orbitron
// Стиль Go / Rust — данные и методы раздельно
struct Point { x: int, y: int }
impl Point {
    public fn dist_sq(self): int { return self.x**2 + self.y**2; }
}
var p = Point { x: 3, y: 4 };  // структурный литерал, без new

// Стиль Java / C# — данные и методы вместе, явный конструктор
class Counter {
    private val: int,
    init(start: int) { self.val = start; }
    public fn inc(self) { self.val += 1; }
    public fn get(self): int { return self.val; }
}
var c = new Counter(0);
```

Используйте тот стиль, который лучше подходит для вашей задачи.

### 3. Прямой доступ к системе

Когда нужна производительность или прямой доступ к ОС, Orbitron не ставит
барьеров:

```orbitron
// Прямые системные вызовы Linux
syscall(SYS_WRITE, STDOUT, buf, n);

// Арифметика указателей
var addr = &x;
ptr_write(addr, 42);

// Внешние C-функции
extern func malloc(size: int): int;
```

### 4. Батарейки в комплекте

Стандартная библиотека покрывает самые распространённые потребности:

| Модуль | Содержимое |
|--------|------------|
| **std/math** | Математические функции и константы |
| **std/bits** | Побитовые операции |
| **std/algo** | Алгоритмические утилиты |
| **std/sys** | Системные вызовы Linux |
| **std/net** | TCP/UDP-сетевое программирование |
| **std/db** | Доступ к базе данных SQLite3 |

## Ключевые возможности на одном экране

```orbitron
import "std/math";

// Константы и типы
const LIMIT: int = 1000;
var pi: float = 3.14;

// Строковая интерполяция
var n = 42;
println($"Ответ: {n}");

// Все виды циклов
for i in 0..10 { println(i); }
repeat 3 { println("привет"); }
while (n > 0) { n -= 1; }

// Сопоставление с образцом
enum Статус { Ок, Ошибка, Ожидание }
var s = Статус.Ок;
match s {
    Статус.Ок      => { println("успех");   }
    Статус.Ошибка  => { println("ошибка");  }
    _              => { println("ожидание");}
}

// Функциональный стиль с конвейером
func double(x: int): int { return x * 2; }
func inc(x: int):    int { return x + 1; }
var result = 5 |> double |> inc;  // 11

// defer — всегда выполняется перед return
func open_resource() {
    defer println("ресурс закрыт");
    println("ресурс открыт");
}

// Оператор возведения в степень
var x = 2 ** 10;   // 1024

// Тернарный оператор
var max = a > b ? a : b;

// unless — инвертированное условие
unless (n == 0) {
    println(100 / n);
}
```

## Что можно создавать?

Orbitron подходит для:

- **Консольных инструментов** — быстрые нативные бинарники с минимальными накладными расходами
- **Алгоритмических программ** — числовые вычисления, обработка данных
- **Системного программирования** — низкоуровневые операции с указателями и системными вызовами
- **Сетевых программ** — TCP/UDP клиенты и серверы (через std/net)
- **Работы с базами данных** — интеграция с SQLite3 (через std/db)
- **Кросс-платформенных JAR** — целевой бэкенд JVM для максимальной переносимости
- **Обучения** — синтаксис чистый и понятный для новичков

## Язык вкратце

| Аспект | Детали |
|--------|--------|
| Расширение файлов | `.ot` |
| Типизация | Статическая (int, float) |
| Память | Стек по умолчанию; куча через sys_alloc / extern malloc |
| Бэкенды | LLVM (нативный бинарник), JVM (.jar) |
| Стандартная библиотека | Написана на самом Orbitron |
| Конкурентность | Пока не реализована (планируется) |
| Строки | Только для вывода; строкового типа пока нет |
| Null / None | Отсутствует |
| Дженерики | Пока не реализованы (планируются) |
