# Глава 8 — Специальные возможности

Orbitron содержит 10 специальных возможностей языка, каждая позаимствована из
разных традиций программирования. В этой главе каждая из них разбирается подробно.

---

## Возможность 1 — Константы (`const`)
*Вдохновлено Rust и C++*

Константы — это неизменяемые именованные значения:

```orbitron
const MAX_PLAYERS:  int   = 100;
const TAX_RATE:     float = 0.15;
const GRID_WIDTH:   int   = 64;
const GRID_HEIGHT:  int   = 64;
```

### Свойства

- Объявляются с `const`, не с `var`
- Требуется аннотация типа: `const N: int = 5;`
- Значение должно быть **литералом** (не вычисляемым выражением)
- Доступны везде в файле и в файлах, которые импортируют этот
- Не могут быть переприсвоены

### Константы vs переменные

```orbitron
const MAX: int = 100;
var count = 0;

// MAX = 50;  // ОШИБКА — константы нельзя переприсваивать
count = 50;   // ОК — переменные можно
```

### Когда использовать константы

Используйте `const` для значений, которые:
- Не изменяются во время работы программы (конфигурация, ограничения)
- Используются в нескольких местах (принцип DRY)
- Делают код более читаемым, давая имя «магическому числу»

```orbitron
// Плохо: магические числа
if (score > 90) { println("Отлично"); }

// Хорошо: именованная константа
const EXCELLENT_SCORE: int = 90;
if (score > EXCELLENT_SCORE) { println("Отлично"); }
```

## Возможность 2 — Оператор возведения в степень (`**`)
*Вдохновлено Python*

Оператор `**` вычисляет возведение в степень:

```orbitron
println(2 ** 10);    // 1024
println(3 ** 4);     // 81
println(10 ** 6);    // 1000000
println(2 ** 0);     // 1
println(2 ** 1);     // 2
```

### Вещественные степени

Для получения вещественного результата используйте вещественные операнды:

```orbitron
var sqrt2 = 2.0 ** 0.5;    // ≈ 1.4142...
var cbrt8 = 8.0 ** 0.333;  // ≈ 2.0 (кубический корень)
```

### Приоритет

`**` имеет **правую ассоциативность** и высокий приоритет:

```orbitron
var a = 2 ** 3 ** 2;   // = 2 ** (3**2) = 2**9 = 512
var b = -2 ** 2;       // = -(2**2) = -4  (унарный минус применяется после)
var c = (2 ** 3) * 4;  // = 8 * 4 = 32
```

### Техническое примечание

Внутренне `**` вызывает функцию `pow(double, double)` из `libm`.
При целочисленных операндах результат приводится обратно к `int`.

## Возможность 3 — Оператор конвейера (`|>`)
*Вдохновлено Elixir и F#*

Оператор конвейера выстраивает вызовы функций слева направо:

```orbitron
значение |> функция
// эквивалентно: функция(значение)

значение |> f |> g |> h
// эквивалентно: h(g(f(значение)))
```

### Базовое использование

```orbitron
func double(n: int): int { return n * 2; }
func inc(n: int):    int { return n + 1; }
func negate(n: int): int { return -n; }

func main() {
    var r = 5 |> double |> inc |> negate;
    println(r);   // -(5*2+1) = -11
}
```

### Читаемость

Без конвейера:
```orbitron
var r = negate(inc(double(5)));  // читать справа налево, изнутри наружу
```

С конвейером:
```orbitron
var r = 5 |> double |> inc |> negate;  // читать слева направо, данные текут естественно
```

### Конвейер с дополнительными аргументами

```orbitron
func add(a: int, b: int): int { return a + b; }
func mul(a: int, b: int): int { return a * b; }

var r = 5 |> add(3) |> mul(2);   // mul(add(5, 3), 2) = 16
println(r);
```

## Возможность 4 — unless
*Вдохновлено Ruby*

`unless (условие)` выполняется, когда условие **ложно**:

```orbitron
unless (условие) {
    // выполняется, когда условие ложно
}
```

### Зачем unless?

```orbitron
// С if — читается неловко из-за отрицания
if (!is_logged_in) { redirect(); }

// С unless — читается естественно
unless (is_logged_in) { redirect(); }
```

### Примеры

```orbitron
var divisor = 5;
unless (divisor == 0) {
    println(100 / divisor);   // безопасное деление
}

var is_empty = 0;
unless (is_empty) {
    println("Обрабатываем данные");
}
```

> `unless` не поддерживает `else`. Для сложного ветвления используйте `if/else`.

## Возможность 5 — Строковая интерполяция (`$"..."`)
*Вдохновлено C# и Kotlin*

Встраивайте значения переменных прямо в строку:

```orbitron
var x = 42;
println($"Ответ: {x}");   // Ответ: 42
```

### Несколько значений

```orbitron
var player = 7;
var score  = 150;
var level  = 3;
println($"Игрок {player}: очки={score}, уровень={level}");
// Игрок 7: очки=150, уровень=3
```

### Интерполяция чисел с плавающей точкой

```orbitron
var pi: float = 3.14159;
println($"Пи приблизительно равно {pi}");   // Пи приблизительно равно 3.141590
```

### Константы в интерполяции

```orbitron
const VERSION: int = 2;
const BUILD:   int = 417;
println($"Orbitron v{VERSION} сборка {BUILD}");
```

### Ограничения

- Только **имена** переменных/констант внутри `{...}` — не выражения
- Только типы `int` и `float`
- Только внутри `println()`

```orbitron
var a = 5;
var b = 3;
var sum = a + b;          // сначала вычислить
println($"Сумма: {sum}"); // затем интерполировать — ОК
// println($"Сумма: {a+b}"); // ОШИБКА — выражения не поддерживаются
```

## Возможность 7 — Перечисления (enum)

## Возможность 8 — Defer
*Вдохновлено Go*

`defer` регистрирует инструкцию для выполнения **непосредственно перед выходом
из функции**, независимо от того, через какой `return` происходит выход:

```orbitron
func process() {
    defer println("Завершено — очистка");
    println("Начало работы");
    // ... работа ...
    println("Конец работы");
}
// Вывод: Начало работы → Конец работы → Завершено — очистка
```

### Несколько defer (порядок LIFO)

При нескольких `defer` они выполняются в порядке **последний-вошёл-первый-вышел**
(как стек):

```orbitron
func multi_defer() {
    defer println("Третий");
    defer println("Второй");
    defer println("Первый");
    println("Тело");
}
// Вывод: Тело → Первый → Второй → Третий
```

### Ранний return

`defer` выполняется даже при раннем выходе из функции:

```orbitron
func guarded(n: int): int {
    defer println("очистка выполняется всегда");

    if (n < 0) {
        println("неверный ввод");
        return -1;   // defer всё равно сработает перед этим return
    }

    println("обработка");
    return n * 2;
}
```

### Практическое применение

```orbitron
func open_resource() {
    // Гарантируем, что close всегда выполнится после open
    defer println("ресурс закрыт");
    println("ресурс открыт");
    // ... работа с ресурсом ...
}
```

> **Примечание для JVM:** В JVM-бэкенде defer реализуется через блок
> `try-finally`, что обеспечивает ту же семантику.

## Возможность 9 — repeat N
*Вдохновлено Lua и Pascal*

Выполнить блок ровно N раз без переменной цикла:

```orbitron
repeat N {
    // тело выполняется N раз
}
```

### Примеры

```orbitron
repeat 5 {
    println("Привет!");
}
// выводит "Привет!" ровно 5 раз
```

```orbitron
var total = 0;
repeat 10 {
    total += 1;
}
println(total);   // 10
```

### repeat с переменным счётчиком

```orbitron
var n = 7;
repeat n {
    println("повтор");
}
```

### repeat vs for

```orbitron
// Эквивалентно — используйте repeat, когда индекс не нужен
repeat 5 { do_something(); }
for _i in 0..5 { do_something(); }
```

Используйте `repeat`, когда:
- Переменная счётчика не нужна
- Хотите подчеркнуть количество, а не диапазон
- Код читается более естественно как "сделать это N раз"

## Возможность 10 — Тернарный оператор (`? :`)
*Вдохновлено C и Java*

Встроенное условное выражение:

```orbitron
var result = условие ? значение_если_истина : значение_если_ложь;
```

### Базовые примеры

```orbitron
var x = 10;
var abs_x = x >= 0 ? x : -x;          // абсолютное значение

var a = 7;
var b = 3;
var max = a > b ? a : b;              // максимум
var min = a < b ? a : b;              // минимум

var n = 4;
var is_even = n % 2 == 0 ? 1 : 0;    // 1 если чётное
```

### В аргументах функций

```orbitron
println(x > 0 ? 1 : 0);
var sign = x > 0 ? 1 : x < 0 ? -1 : 0;   // функция знака
```

### Право-ассоциативные цепочки

```orbitron
var score = 82;
var grade = score >= 90 ? 4 :
            score >= 80 ? 3 :
            score >= 70 ? 2 :
            score >= 60 ? 1 : 0;
```

### Техническое примечание

В LLVM-бэкенде тернарный оператор компилируется в phi-узел (не в инструкцию `select`),
поэтому обе ветви могут иметь побочные эффекты.

## 8.9 — Все 10 возможностей вместе

Следующий пример демонстрирует все 10 возможностей в одной программе:

```orbitron
// examples/07_advanced/features.ot

const MAX_SCORE: int = 100;    // Возможность 1: const

func double(n: int): int { return n * 2; }
func inc(n: int):    int { return n + 1; }

enum Grade { Fail, Pass, Good, Excellent }  // Возможность 7: enum

func get_grade(score: int): int {
    return score >= 90 ? Grade.Excellent :  // Возможность 10: тернарный
           score >= 70 ? Grade.Good     :
           score >= 50 ? Grade.Pass     : Grade.Fail;
}

func compute(base: int): int {
    defer println("вычисление завершено");   // Возможность 8: defer
    var power = base ** 3;                   // Возможность 2: **
    return power |> double |> inc;           // Возможность 3: |>
}

func main() {
    var scores = [75, 90, 45, 82, 55];      // Возможность 6: массивы

    for i in 0..5 {
        var s = scores[i];

        unless (s > MAX_SCORE) {            // Возможность 4: unless
            var g = get_grade(s);
            println($"Балл {s} => оценка {g}");  // Возможность 5: интерполяция

            match g {
                Grade.Excellent => { println("Отлично!"); }
                Grade.Good      => { println("Хорошо");   }
                Grade.Pass      => { println("Зачтено");  }
                _               => { println("Не сдано"); }
            }
        }
    }

    var result = compute(3);
    println(result);    // (3**3)*2+1 = 55

    repeat 3 {          // Возможность 9: repeat
        println("готово");
    }
}
```
