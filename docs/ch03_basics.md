# Глава 3 — Основы языка

В этой главе рассматриваются фундаментальные строительные блоки любой программы на Orbitron:
переменные, типы, константы, операторы и строки.

---

## 3.1 — Переменные

Orbitron различает **иммутабельные** (`let`) и **мутабельные** (`mut`) переменные:

```orbitron
let x = 42;      // иммутабельная — нельзя переприсвоить
mut y = 10;      // мутабельная — можно переприсваивать
```

Тип переменной определяется по начальному значению:
- Целочисленные литералы → `i64` (64-битное знаковое целое)
- Литералы с плавающей точкой → `f64` (64-битное двойной точности)
- `true` / `false` → `i64` (1 и 0 соответственно)

Псевдонимы `int` и `float` сохраняются для совместимости, но рекомендуется использовать `i64`/`f64`.

### Аннотации типов (необязательно)

```orbitron
let count:  i64 = 0;
let ratio:  f64 = 1.5;
mut speed:  i64 = 300;
```

### Переприсвоение

Только `mut`-переменные допускают переприсвоение:

```orbitron
mut score = 0;
score = 100;
score = score + 50;
```

Попытка переприсвоить `let`-переменную — ошибка.

### Правила именования переменных

- Должно начинаться с буквы или подчёркивания (`_`)
- Может содержать буквы, цифры и подчёркивания
- Чувствительно к регистру: `count` и `Count` — разные переменные
- Только ASCII (кириллица в именах переменных не поддерживается)
- Не может совпадать с ключевым словом

```orbitron
var total_score = 0;    // ОК
var _temp       = 99;   // ОК — ведущее подчёркивание допустимо
var x2          = 10;   // ОК
// var 2x = 10;         // ОШИБКА — начинается с цифры
```

## 3.2 — Константы

Константы объявляются директивой `#const`. Значение должно быть числовым литералом:

```orbitron
#const MAX_SIZE:   i64 = 1000;
#const TAX_RATE:   f64 = 0.2;
#const SCREEN_W:   i64 = 1920;
#const SCREEN_H:   i64 = 1080;
```

### Свойства констант

- **Неизменяемы** — нельзя переприсвоить после объявления
- **Любая область видимости** — можно объявлять на верхнем уровне или внутри функции
- **Видны везде** — константы верхнего уровня доступны всем функциям в том же файле и в файлах, которые его импортируют
- **Значение должно быть литералом** — не может быть вычисляемым выражением вроде `2 * 512`

```orbitron
#const MAX: i64 = 100;

fn main() {
    #const LOCAL: i64 = 50;   // константа уровня функции
    println!(MAX);             // 100
    println!(LOCAL);           // 50
}
```

> **Примечание:** Для настоящих значений π и e в виде `f64` используйте
> `#import "std/math"` — там они вычислены на этапе выполнения.

## 3.3 — Типы данных

В Orbitron два примитивных числовых типа:

| Тип | Псевдоним | Размер | Диапазон | Примеры |
|-----|-----------|--------|----------|---------|
| `i64` | `int` | 64-бит знаковый | −9.2 × 10¹⁸ до +9.2 × 10¹⁸ | `0`, `42`, `-100` |
| `f64` | `float` | 64-бит двойной точности | ≈ ±1.8 × 10³⁰⁸ | `3.14`, `2.718`, `-0.5` |

Рекомендуется использовать `i64`/`f64` в новом коде.

### Булевы значения

В Orbitron нет отдельного типа `bool`. Булевы значения представляются как `int`:

| Литерал | Значение |
|---------|---------|
| `true`  | `1`     |
| `false` | `0`     |

```orbitron
let flag = true;    // flag == 1
let done = false;   // done == 0

if (flag) {
    println!("flag истинно");
}
```

Операторы сравнения тоже возвращают `i64`:

```orbitron
let result = 5 > 3;   // result == 1 (не булево, просто i64 1)
```

### Повышение типа

Когда в одном арифметическом выражении встречаются `i64` и `f64`,
`i64` автоматически повышается до `f64`:

```orbitron
let a: i64 = 10;
let b: f64 = 3.0;
let c = a + b;   // c — f64 (13.0), a был повышен
```

## 3.4 — Арифметические операторы

| Оператор | Значение | Пример | Результат |
|----------|----------|--------|-----------|
| `+` | Сложение | `3 + 4` | `7` |
| `-` | Вычитание | `10 - 3` | `7` |
| `*` | Умножение | `6 * 7` | `42` |
| `/` | Деление | `10 / 3` | `3` (целочисленное) |
| `%` | Остаток | `10 % 3` | `1` |
| `**` | Возведение в степень | `2 ** 10` | `1024` |
| `-` (унарный) | Отрицание | `-x` | отрицает x |

### Поведение деления

Деление двух целых — **целочисленное** (усекает дробную часть):

```orbitron
var a = 10 / 3;    // 3, а не 3.333...
var b = 7 / 2;     // 3
var c = -7 / 2;    // -3 (округление к нулю)
```

Для деления с плавающей точкой используйте вещественные литералы:

```orbitron
var d = 10.0 / 3.0;   // 3.333...
var e = 7 / 2.0;      // 3.5  (int повышается до float)
```

### Оператор возведения в степень `**`

```orbitron
let p1 = 2 ** 10;    // 1024
let p2 = 3 ** 4;     // 81
let p3 = 2.0 ** 0.5; // ≈ 1.414 (квадратный корень как f64)
```

Внутри `**` вызывает функцию `pow()` из `libm`. При целочисленных операндах
результат приводится обратно к `int`.

---

## 3.5 — Операторы сравнения

Сравнение возвращает `1` (истина) или `0` (ложь):

| Оператор | Значение | Пример | Результат |
|----------|----------|--------|-----------|
| `==` | Равно | `5 == 5` | `1` |
| `!=` | Не равно | `5 != 3` | `1` |
| `<` | Меньше | `3 < 5` | `1` |
| `<=` | Меньше или равно | `5 <= 5` | `1` |
| `>` | Больше | `7 > 3` | `1` |
| `>=` | Больше или равно | `4 >= 4` | `1` |

```orbitron
let x = 42;
let is_positive = x > 0;      // 1
let is_even     = x % 2 == 0; // 1
let is_big      = x >= 100;   // 0
```

---

## 3.6 — Логические операторы

| Оператор | Значение | Пример |
|----------|----------|--------|
| `&&` | Логическое И | `a > 0 && b > 0` |
| `\|\|` | Логическое ИЛИ | `a == 0 \|\| b == 0` |
| `!` | Логическое НЕ | `!flag` |

Любое ненулевое значение считается истинным:

```orbitron
let a = 5;
let b = 0;

if (a > 0 && b == 0) {
    println!("a положительное и b равно нулю");
}

if (a == 0 || b == 0) {
    println!("хотя бы одно равно нулю");
}

if (!b) {
    println!("b ложное (ноль)");
}
```

## 3.7 — Составное присваивание

| Оператор | Эквивалент |
|----------|-----------|
| `x += n` | `x = x + n` |
| `x -= n` | `x = x - n` |
| `x *= n` | `x = x * n` |
| `x /= n` | `x = x / n` |
| `x %= n` | `x = x % n` |
| `x ^= n` | `x = x ^ n` |

```orbitron
mut total = 0;
total += 10;   // total == 10
total += 20;   // total == 30
total -= 5;    // total == 25
total *= 2;    // total == 50
total /= 5;    // total == 10

mut n = 17;
n %= 5;        // n == 2  (остаток от деления 17 на 5)

mut flags = 0b1010;
flags ^= 0b1100;   // flags == 0b0110  (XOR — переключение битов)
```

> **Побитовые операторы** `%=` и `^=` описаны подробнее в [Главе 8](ch08_features.md).

---

## 3.8 — Приоритет операторов

От наименьшего к наибольшему:

| Уровень | Операторы | Ассоциативность |
|---------|-----------|-----------------|
| 1 (наименьший) | `\|>` | левая |
| 2 | `? :` (тернарный) | правая |
| 3 | `\|\|` | левая |
| 4 | `&&` | левая |
| 5 | `==` `!=` `<` `<=` `>` `>=` | — |
| 6 | `+` `-` | левая |
| 7 | `*` `/` `%` | левая |
| 8 | `-` `!` `~` (унарные) | правая |
| 9 | `**` | правая |
| 10 (наибольший) | `.поле` `.метод()` `[индекс]` | левая |

При сомнениях используйте скобки:

```orbitron
var a = 2 + 3 * 4;       // 14 (не 20) — * имеет приоритет выше
var b = (2 + 3) * 4;     // 20
var c = 2 ** 3 ** 2;     // 2 ** (3**2) = 2**9 = 512 (право-ассоциативно)
```

## 3.9 — Строковые литералы

Строки в Orbitron предназначены только для вывода — они могут появляться
только внутри `println()`. Типа данных "строка" для хранения в переменной пока нет.

```orbitron
println("Привет, мир!");
println("Строка с \"экранированными\" кавычками");
println("Табуляция:\tздесь");
```

Стандартные экранирующие последовательности: `\"`, `\\`, `\n`, `\t`.

---

## 3.10 — Строковая интерполяция

Синтаксис `"\{...}"` позволяет встраивать значения переменных прямо в строку.
Это основной способ вывода вычисленных значений:

```orbitron
let x = 42;
let pi: f64 = 3.14;
#const VER: i64 = 2;

println!("x = \{x}");         // x = 42
println!("pi = \{pi}");       // pi = 3.140000
println!("версия: \{VER}");   // версия: 2
```

### Правила интерполяции

- Интерполировать можно переменные и константы типов `i64` и `f64`
- Синтаксис: `\{имя_переменной}` внутри строки `"..."`
- Можно использовать только внутри `println!()`
- В одной строке допускается несколько вставок

```orbitron
let a = 10;
let b = 20;
let sum = a + b;
println!("a=\{a}, b=\{b}, сумма=\{sum}");   // a=10, b=20, сумма=30
```

> **Ограничение:** Выражения типа `\{a + b}` внутри интерполяции не поддерживаются —
> только имена переменных или констант. Сначала вычислите значение в переменную,
> затем интерполируйте.

## 3.11 — Кортежи

Кортеж — это значение, содержащее несколько элементов. Используйте синтаксис `(a, b)`:

```orbitron
let point = (10, 20);
```

### Распаковка кортежа

Чтобы извлечь значения из кортежа, используйте деструктурирующее объявление:

```orbitron
let (x, y) = (10, 20);
println!(x);   // 10
println!(y);   // 20
```

### Обмен значений через кортеж

```orbitron
mut a = 5;
mut b = 9;
let (a, b) = (b, a);   // классический обмен
println!(a);   // 9
println!(b);   // 5
```

### Кортеж как результат функции

Кортежи удобны для параллельного присвоения нескольких значений:

```orbitron
fn main() {
    let (lo, hi) = (3, 7);
    println!("мин=\{lo}, макс=\{hi}");   // мин=3, макс=7
}
```

> **Примечание:** Кортежи в Orbitron — это прежде всего синтаксическая конструкция
> для параллельного присвоения. Они не являются отдельным типом данных первого класса.

## 3.12 — Аннотации (`@`)

Аннотации — это метаданные, которые можно прикреплять к функциям или классам
с помощью префикса `@`. Они не изменяют поведение программы, но несут
дополнительную информацию для читателя или инструментов:

```orbitron
@deprecated
fn old_api(x: i64): i64 {
    return x * 2;
}

@test
fn test_add() {
    let r = add(2, 3);
    assert_eq!(r, 5);
}

@inline
fn fast_double(x: i64): i64 => x * 2;
```

### Предопределённые аннотации

| Аннотация | Назначение |
|-----------|-----------|
| `@test` | Помечает функцию как тест |
| `@deprecated` | Помечает устаревший API |
| `@inline` | Подсказка компилятору для инлайнинга |
| `@override` | Указывает на переопределение метода |

```orbitron
class Animal {
    pub fn speak(self) {
        println!("...");
    }
}

class Dog extends Animal {
    @override
    pub fn speak(self) {
        println!("Гав!");
    }
}
```

> **Примечание:** Компилятор фиксирует аннотации в AST, но пока не применяет
> к ним специальную логику — они служат документацией и меткой намерений.

## 3.13 — Комментарии

Комментарии игнорируются компилятором:

```orbitron
// Однострочный комментарий

/*
 * Многострочный блочный комментарий.
 * Можно писать сколько угодно строк.
 */

var x = 42;  // встроенный комментарий после кода
```

## Итоговая таблица

| Конструкция | Синтаксис | Примечание |
|-------------|-----------|-----------|
| Иммутабельная переменная | `let name = value;` | Нельзя переприсвоить |
| Мутабельная переменная | `mut name = value;` | Можно переприсвоить |
| С аннотацией типа | `let name: i64 = value;` | Необязательно |
| Константа | `#const NAME: i64 = value;` | Неизменяемая, только литерал |
| Переприсвоение | `name = new_value;` | Только для `mut` |
| Составное присваивание | `x += 5;` | Также `-=`, `*=`, `/=`, `%=`, `^=` |
| Кортеж | `let (a, b) = (1, 2);` | Деструктурирующее присвоение |
| Аннотация | `@test`, `@deprecated` | Метаданные для функций/классов |
| Строка | `println!("текст");` | Только для вывода |
| Интерполяция | `println!("x=\{x}");` | Встраивает i64 и f64 |
| Комментарий | `// строка` или `/* блок */` | Игнорируется |

## Пример: всё вместе

```orbitron
// examples/01_basics/variables.ot
// Демонстрация переменных, констант, типов и операторов.

#const GRAVITY: i64 = 10;    // ускорение свободного падения, м/с²
#const MASS:    i64 = 75;    // масса, кг

fn main() {
    // Переменные
    let height:   i64 = 100;   // метры
    mut velocity: i64 = 0;

    // Арифметика
    let force  = MASS * GRAVITY;            // 750 Н
    let energy = MASS * GRAVITY * height;   // 75000 Дж

    // Составное присваивание
    velocity += 30;

    // Остаток и XOR через составное присваивание
    mut n = 17;
    n %= 5;    // n == 2
    mut bits = 0b1010;
    bits ^= 0b0011;   // bits == 0b1001

    // Кортеж — параллельное присвоение
    let (min_v, max_v) = (0, 100);
    println!("диапазон: \{min_v}..\{max_v}");   // диапазон: 0..100

    // Оператор степени
    let v_sq = velocity ** 2;   // 900

    // Сравнение
    let is_moving = velocity > 0;   // 1

    // Интерполяция
    println!("Сила: \{force} Н");
    println!("Энергия: \{energy} Дж");
    println!("Скорость: \{velocity} м/с");
    println!("Скорость в квадрате: \{v_sq}");
    println!("В движении: \{is_moving}");
}
```
